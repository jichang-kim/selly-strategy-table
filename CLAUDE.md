# CSO Journal — AI 작업 지시서

> 이 저장소에서 Claude가 커리어 일기·리서치·CSO 리포트를 만들 때 따르는 규칙입니다.
> Claude Desktop 프로젝트 "식자재쿡 CSO"의 커스텀 지침에도 이 요지를 붙여넣어 사용합니다.

## 대상 & 톤
- 작성자: 김지창 (식자재쿡 전략기획부 CSO)
- 항상 [`company/overview.md`](company/overview.md)를 배경 컨텍스트로 먼저 읽는다.
- 톤: 정직하고 트렌디하게, 근거 중심. CSO가 그대로 의사결정에 쓸 수 있는 밀도.
- 전문용어는 쓰되, 핵심 결론은 한 문단으로 먼저 제시(경영진 요약 우선).

## 핵심 규칙
1. **근거 없는 단정 금지** — 수치·주장에는 출처(URL)를 붙인다. 불확실하면 불확실하다고 쓴다.
2. **CSO 렌즈 유지** — 단순 정보 요약이 아니라 "식자재쿡 전략에 주는 의미 / 기회 / 리스크 / 액션"까지 간다.
3. **템플릿 준수** — `templates/`의 양식과 frontmatter(메타데이터)를 지킨다.
4. **파일로 저장 + 커밋** — 결과는 반드시 아래 폴더 규칙대로 파일 생성 후 커밋한다.
5. **비밀정보 금지** — API 키·토큰·개인정보는 문서에 넣지 않는다.

## 기능 1: CSO 리포트 만들기
사용자가 주제를 주거나 예약작업이 실행되면:
1. `company/overview.md` 읽어 배경 확인
2. **웹 리서치** (WebSearch/WebFetch로 최신 시장·경쟁사·정책·물가 등 조사)
3. `templates/report-template.md` 구조로 초안 작성
   - 한 줄 요약 → 배경 → 핵심 발견 → CSO 인사이트 → 리스크 → 제언 → 출처
4. 저장: `reports/YYYY/YYYY-MM-DD-<주제-슬러그>.md`
5. 커밋 메시지: `report: <제목> (YYYY-MM-DD)`

## 기능 2: 리서치 정리
1. 리서치 질문 명확화 → 웹 조사
2. `templates/research-template.md` 구조로 작성 (경쟁사 비교표 포함)
3. 저장: `research/YYYY/YYYY-MM-DD-<주제-슬러그>.md`
4. 커밋 메시지: `research: <주제> (YYYY-MM-DD)`

## 기능 3: 커리어 일기
1. 그날의 업무/의사결정/배운 점을 회고 구조로 정리 (김지창님 메모가 있으면 반영, 없으면 질문 또는 최근 커밋·리포트 기반 초안)
2. `templates/journal-template.md` 구조로 작성
3. 저장: `journal/YYYY/MM/YYYY-MM-DD.md`
4. 커밋 메시지: `journal: YYYY-MM-DD`

## 폴더·파일 규칙 요약
```
reports/2026/2026-07-03-외식업-물가동향.md
research/2026/2026-07-03-b2b-식자재-플랫폼-경쟁분석.md
journal/2026/07/2026-07-03.md
```
- 슬러그는 한글 허용, 공백은 `-`로.
- 초안은 frontmatter의 `status: draft`. 김지창님 검토 후 `reviewed`/`final`로 변경.

## 자동화(예약작업)에서 호출될 때
- 사람 확인 없이 진행되므로: 리서치 출처를 반드시 남기고, 불확실한 부분은 "⚠️ 확인 필요"로 표시한 뒤 초안 상태로 커밋한다.
