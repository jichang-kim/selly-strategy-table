# K&K 글로벌 트레이딩 (K-Market) — 개발 제안
{: .no_toc }

## 한 줄 요약

K-Market은 이미 앱(K-Market Mall)·150개 매장·온·오프 통합 포인트를 보유하고 있습니다. 따라서 제안의 핵심은 "신규 구축"이 아니라 **기존 자산의 고도화 · 시스템 통합 · 데이터화**이며, 식자재쿡은 2년+ 프로덕션 운영 중인 B2B 커머스 스택을 현지화하여 빠르게 이식할 수 있습니다. 내부 시스템(POS·ERP·CRM)이 비공개이므로 **① 진단(Phase 0) → ② 모듈 메뉴 → ③ 3단계 패키지** 구조로 어떤 상황에서든 유연하게 제안합니다.

**CSO 인사이트** — 정보가 적은 상대에게는 "정답 제시"보다 "as-is 데모 요청 + 함께 설계" 프레이밍이 유리합니다. 소액·단기 진단으로 신뢰를 쌓고 본계약으로 잇는 것이 양측 리스크를 줄입니다.

## 1. K-Market 분석

| 지표 | 값 |
|---|---|
| 베트남 매장 수 | **150+** (2025.11) |
| 임직원 | 약 2,000명 |
| 매출 중 B2B 비중 | 약 60% |
| 취급 SKU | 15,000+ |

- **회사·규모**: 2002년 진출, 고상구 회장·신영화 President, 하노이 본사(물류센터 병설). "베트남 최대 한국식품 유통기업"·베트남 Top100 브랜드 다년 선정. 협력사 600~1,000+(호텔·레스토랑·급식·로컬마트), 삼성·메리어트 등 B2B 납품. '5년 내 매출 1조' 목표(회장 발언, 목표치).
- **디지털**: K-Market Mall 앱 운영(iOS/Android). 온라인 쇼핑 + 자택배송, 온·오프 통합 1% 포인트·기프트카드, 부가서비스(휴대폰 충전·공과금). ShopeeFood 입점, FB·IG·TikTok 채널.
- **물류·전략**: 회장 **3A 전략(Anywhere·Anytime·Anyone)** = 옴니채널 지향. 호치민(동나이) **첨단 복합물류센터 1억 달러 투자**(상온·냉장·냉동 풀필먼트·3PL·해외직구 물류·남북 통합망).
- **시장·규제**: 결제 MoMo·ZaloPay·VNPay-QR. 경쟁 롯데마트·GS25(414개)·AEON·WinMart. 외국계 소매 **ENT(경제적수요심사)**, **전자상거래법 2025**(2026.7 시행).

**핵심 시사점**: ① 앱은 이미 존재 → '리뉴얼·통합' 앵글. ② 3A 전략·1억$ 물류 투자 = 옴니채널·풀필먼트·데이터 통합 수요(가설). ③ 내부 기간계 비공개 → **진단부터** 시작이 안전.

> ⚠️ 확인 필요: 재무(매출) 수치, POS/ERP/CRM 벤더·자체개발 여부, 직영/가맹 비율은 공개 자료에 없어 실사 필요.

## 2. 개발 니즈 가설 (실사로 검증)

1. 온라인 주문·퀵커머스 고도화 (앱 리뉴얼·근거리 즉시배송·채널 재고 동기화)
2. 멤버십·CRM 통합 (통합 회원·등급/쿠폰·타겟 마케팅)
3. POS–재고–ERP 통합 (매장 POS↔본사 재고 실시간·발주 자동화)
4. WMS·풀필먼트·콜드체인 IT (신물류센터 오더 라우팅·온도 IoT)
5. B2B 도매 주문 포털 (거래처 단가·견적·여신·정산 — 매출 60%)
6. 로컬 결제 통합 (MoMo·ZaloPay·VNPay-QR·카드)
7. PB·트레이스빌리티 (원산지·인증 이력·상품 마스터)
8. BI·옴니채널 데이터 (통합 분석·수요예측, 3A 백엔드)

## 3. Why 식자재쿡 — '이미 만든 것 = 바로 제안 가능한 것'

| 보유 자산 | 설명 | K-Market 이식성 |
|---|---|---|
| 커머스 주문앱(iOS/Android) | 4만+ SKU·검색·즐겨찾기·소셜로그인·푸시 | 높음(즉시) |
| 화이트라벨·지점 구조 | 식자재쿡→브랜드쿡 파생, 지점별 공지 | 높음 |
| 거래처별 단가·전용코드 | 고객사별 상품 마스터·단가(B2B 핵심) | 높음 |
| 자체페이·포인트·예치금 | 다중 PG·등급 적립(현지 PG 교체) | 중 |
| 배송추적·사진·알림 | 콜드체인 배송 SW 레이어 | 중 |
| 관리자 ERP·대시보드 | 발주·정산·거래처(실물 미확인 → 데모 검증) | ⚠️ 검증 필요 |

**파트너십 각도**: 고상구 회장은 세계한상 네트워크 중심·한국 지자체(고양시 등) 협력에 적극적 → 한국계 검증 IT 파트너에 우호적일 가능성(가설). 'K-Food 유통을 아는 개발사' 도메인 적합성을 전면에.

---

## 4. 제안 접근 — 진단부터, 모듈형으로

- **Phase 0 · Discovery(as-is 진단, 2~4주)**: 현행 앱·POS·ERP·물류·회원 실사, 연동 지점·우선순위·ROI 정의, PoC 범위 확정. 산출물 = as-is 진단서·to-be 아키텍처·확정견적.
- **Phase 1~ · 모듈형 실행**: 4대 도메인 모듈을 상황에 맞게 조립. API-우선 설계로 독립 배포, 현지 파트너(운영·QA·현지화)와 브릿지 협업.

## 5. 제안 모듈 메뉴 (4대 도메인 · 공수 M/M)

### A. 모바일 커머스 앱

| 모듈 | 내용 | 공수(M/M) |
|---|---|---|
| A1 리테일 앱 리뉴얼(B2C) | 다국어·UX·현지 PG | 6~9 |
| A2 퀵커머스·근거리 배송 | 즉시배송·슬롯·추적 | 3~5 |
| A3 B2B 도매 포털 | 거래처 단가·여신·정산 | 5~8 |
| A4 멤버십·포인트 통합 | 통합 포인트·등급·쿠폰 | 3~5 |

### B. ERP · 전산

| 모듈 | 내용 | 공수(M/M) |
|---|---|---|
| B1 상품마스터·SKU | 다국어 카탈로그·PB | 2~3 |
| B2 발주·재고 | 입출고·유통기한·자동발주 | 4~6 |
| B3 거래처·정산·여신 | 계약가·예치금·e-invoice | 3~5 |
| B4 관리자·권한 | RBAC·승인 워크플로우 | 2~4 |

### C. POS · 리테일 통합

| 모듈 | 내용 | 공수(M/M) |
|---|---|---|
| C1 POS↔ERP 연동 | 실시간 재고 미들웨어 | 4~7 |
| C2 신선·유통기한 | FIFO·로스 리포트 | 2~3 |
| C3 매장 발주 자동화 | 자동발주·전자라벨·태블릿 | 2~4 |
| C4 로컬 결제 통합 | MoMo·ZaloPay·VNPay-QR | 2~4 |

### D. 물류 · 풀필먼트 · CRM · 데이터

| 모듈 | 내용 | 공수(M/M) |
|---|---|---|
| D1 WMS·풀필먼트 | 피킹·패킹·오더 라우팅·3PL | 5~8 |
| D2 콜드체인 IoT | 온도수집·알림·대시보드 | 2~4 |
| D3 옴니채널 CRM | 세그먼트·캠페인·쿠폰엔진 | 4~6 |
| D4 BI·수요예측 | 대시보드·재고최적화 | 3~5 |

## 6. 3단계 제안 패키지

| 패키지 | 범위 | 공수·기간 |
|---|---|---|
| **Tier 1 · Quick Win** | 앱 리뉴얼(A1)·멤버십(A4)·로컬 결제(C4)·기본 대시보드 | ≈ 12~18 M/M · 3~4개월 |
| **Tier 2 · Core** (추천 진입점) | Tier1 + 퀵커머스(A2)·B2B 포털(A3)·상품/발주(B1·B2)·거래처 정산(B3) | ≈ 25~35 M/M · 6~9개월 |
| **Tier 3 · Full 옴니채널** | Tier2 + POS–ERP(C1~C3)·WMS/콜드체인(D1·D2)·CRM(D3)·BI(D4) | ≈ 45~60 M/M · 12~15개월 |

모두 Phase 0 진단(2~4주)을 선행하며, 모듈 단위 부분 착수·병렬화로 단축 가능합니다.

## 7. 공수 산정 & 팀

| 도메인 | 대표 모듈 | 공수(M/M) |
|---|---|---|
| A. 커머스 앱 | 앱 리뉴얼·퀵커머스·B2B·멤버십 | 17~27 |
| B. ERP·전산 | 상품마스터·발주/재고·정산·권한 | 11~18 |
| C. POS·리테일 | POS연동·유통기한·자동발주·결제 | 10~18 |
| D. 물류·CRM | WMS·콜드체인·CRM·BI | 14~23 |

**내부 가용 팀**: 9년차 풀스택 리드 · 6·5년차 FE 2명 · 4년차 BE 1명 · 2년차 UX/UI · 외주 디자이너 1 · PM(프로젝트 리드) 1. 엔지니어 실가동 ≈ 4인 → 월 약 4 M/M 병렬 처리.

> ⚠️ 확인 필요: 공수는 유사 커머스/ERP 구축 경험 기반 사전 추정입니다. 연동 대상(POS·WMS·e-invoice)·현지화 범위에 따라 변동하며, 확정 견적은 Phase 0 산출물로 제시합니다.

## 8. 실행 체계 · 리스크

- **협업 모델**: 한국 본팀(설계·개발) + 현지 파트너(운영·QA·현지화·CS) 브릿지. API-우선·모듈형 배포, 데이터 마이그레이션 포함. PM이 K-Market 실무·의사결정 라인과 직접 커뮤니케이션.
- **리스크·대응**: 규제(외국계 소매 ENT·전자상거래법 2025 → 현지 법인/파트너 준수) / 결제·세금(현지 PG·e-invoice → 진단 확정) / 기존 시스템 불확실(POS·ERP 비공개 → Phase 0 해소) / 언어·현지화(한·베·영 → 현지 파트너·외주 디자인).

## 10. 다음 단계

1. **미팅·니즈 확인**: as-is 시스템 데모 요청, 우선 도메인 합의, 관심 Tier 선택.
2. **Phase 0 진단 계약**: 2~4주 유상 진단으로 to-be·확정견적 도출(저위험 시작).
3. **PoC / 본계약**: Tier 1 Quick-Win PoC → 성과 확인 후 확장, MOU 검토.

> 미팅 오프닝(예시): "K-Market Mall과 150개 매장·물류 인프라를 이미 갖추셨으니, 저희는 이를 하나로 통합하고 데이터로 움직이게 만드는 파트너입니다."
