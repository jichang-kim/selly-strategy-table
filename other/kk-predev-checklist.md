# K-Market 개발 착수 전 사전 검토 체크리스트
{: .no_toc }

> 상태: **draft** · K-Market(K&K 글로벌 트레이딩) 개발 제안의 부속 자료. 베트남 현지 규제 리서치 + 식자재쿡 자체 시스템 갭 분석 종합. 상세 법령 근거는 하위 문서 「베트남 규제·결제·세금·데이터 조사 보고서」 참조. 실제 착수·투자 전 현지 로펌·회계법인 자문 필수.

## 0. 먼저 알아야 할 "레드플래그" 6가지

대부분 2025~2026년에 법령이 새로 바뀌어 **최신 기준으로 설계**해야 합니다.

1. **원화(KRW) 현금 수취는 불법.** 베트남은 VND가 유일 법정통화이며 외화 결제·가격 표시가 금지됩니다. "매장에서 원화를 받고 거스름돈을 동화로 준다"는 관행은 외환관리 규정 위반으로, 법인 벌금 최대 2억 VND + 외화 몰수(신 시행령 340/2025, 2026.2.9 시행). → **시스템에 '원화 수취' 기능을 넣는 순간 위법 방조가 될 수 있어, 설계 단계에서 배제하거나 별도 환전 프로세스로 분리해야 함.**
2. **계산대 전자세금계산서(e-invoice)가 법적 의무.** 슈퍼마켓·소매업은 2025.6.1부터 세무당국 실시간 연동형 e-invoice 사용 의무(Decree 70/2025). → 단순 POS로는 불충분, 승인 T-VAN 사업자 연동 아키텍처가 스코프 핵심.
3. **개인정보보호가 '법률'로 격상.** 기존 PDPD(Decree 13/2023)는 2026.1.1 폐지, 신 개인정보보호법(PDPL, Law 91/2025)+Decree 356/2025로 대체. **국외이전 위반 시 매출의 5% 과징금.** → 한국 본사로의 데이터 전송이 최대 리스크.
4. **결제는 현지 라이선스 게이트웨이 경유가 유일한 합법 경로.** VNPAY·MoMo·ZaloPay 등과 K-Market **법인 명의**로 가맹계약 후 SDK/API 연동. **개발사/본사 계좌로 대금 pooling 금지**(무허가 지급결제업 트리거).
5. **전자상거래법·행정구역 전면 개편.** 신 전자상거래법(Law 122/2025) 2026.7.1 발효. 행정구역이 63→34개 성/시로 통합, **군(구)급 완전 폐지(2025.7.1)** → **주소 데이터 모델을 2단계(성/시 → 방/사)로 재설계 필수.**
6. **한국은 CPTPP·EVFTA 비당사국** → 추가 소매점 개설 시 ENT(경제적수요심사) 조약 면제가 자동 적용 안 됨.

## A. 결제 · 정산 (베트남)

- **결제 3계층 커버**: VietQR/NAPAS 247(국가 QR, 저비용) → MoMo·ZaloPay(전자지갑) → Visa/Master(외국인·법인).
- **정산 주기·수수료(MDR)**: VNPAY 국내 ≈ 1.1%+1,650 VND, 해외카드 ≈ 2.75%, 월 15억 VND 초과 T+1. 지갑사는 계약별 상이 → 실측 확인.
- **결제 확정 대사**: VietQR '입금=결제확정' 자동 대사(계좌 조회 API/웹훅)를 POS/ERP에 구현.
- **가맹계약 리드타임**: K-Market 법인 명의 상용계약 후 production 전환 → 프로젝트 초기에 확보.

## B. 통화 · 현금 처리 (원화 이슈)

- **원화 수취 기능 배제/분리**: 매장 결제는 VND만. KRW 결제·거스름돈 로직 미탑재 원칙.
- **환율·회계**: 외화 유입은 '결제'가 아닌 '환전'(별도 라이선스 영역) → 현지 자문 필수.
- **현금 시재 모듈**: 시재/드로어/입출금/마감 정산(식자재쿡 현재 부재).

## C. 세금 · 전자세금계산서

- **T-VAN 연동 e-invoice**: 판매 즉시 세무당국 연동 발행 의무. 승인 T-VAN(VNPT·Viettel·MISA 등) 연동.
- **다중 VAT**: 면세/5%/8%/10% 날짜·품목 기반 엔진. 8% 감면 2026.12.31 일몰 → 2027 자동 10% 복귀.
- **B2B 계산서**: 매출 60% B2B → 거래처별 e-invoice 자동 발행·정산 연계.

## D. 데이터 보호 · 현지화 · 호스팅

- **PDPL 준수**: 목적별 분리 동의, 국외이전 영향평가(CTIA)·A05 통지, DPO 지정. 위반 시 매출 5% 과징금.
- **데이터 현지화**: 이커머스·온라인결제 등 '대상 서비스' 해당 시 베트남 국내 저장(24개월) → 해당 여부 확인 필요.
- **호스팅 리전**: 베트남 현지(AWS 하노이 Local Zone/국내 클라우드) 우선, 본사 원본 이전 최소화.

## E. 법 · 규제 · 법인 · 주소체계

- **ENT·업태**: 500㎡ 미만+승인 상업센터 내 면제 가능(미니슈퍼/편의점 분류 회피). 지분·지주구조 설계.
- **전자상거래법 2025**: 신법(122/2025) 기준, MOIT 신고+확인 마크, 베트남 법인/대리인.
- **주소 데이터 모델**: 군(구) 폐지 → 2단계(성/시→방/사), GSO 공식 코드(성2+사/방5자리).

## F. 우리(식자재쿡) 시스템 갭 — 자체 점검

식자재쿡은 **오프라인 매장·POS·현금계산대가 없는 온라인 B2B 식자재 '중개' 플랫폼**(위탁사 모집 → 거래처 중개·판매)입니다. K-Market은 **다점포 오프라인 소매 + 온라인 + B2B** 리테일 마트라 구조가 다릅니다.

| 영역 | 식자재쿡 보유(재사용 가능) | K-Market에 새로 필요(갭) |
|---|---|---|
| 온라인 주문·커머스 | 앱·카탈로그·장바구니·주문내역 | 다국어·현지 UX |
| B2B·거래처 | 거래처별 단가·전용코드·예치금·정산 | 베트남 VAT·e-invoice 연계 |
| 다자 정산(위탁사 모델) | 공급사-거래처 중개 정산 로직 | 소매 '단일 주체 다점포' 모델로 재설계 |
| 결제 | 쿡페이·가상계좌(한국 PG) | **전면 교체**: VietQR·지갑·카드+T-VAN |
| **오프라인 POS 계산대** | **없음** | 바코드·저울·현금 드로어·영수증·계산대 e-invoice |
| **매장 재고(다점포)** | 없음 | 150개 매장별 재고·이동·실사 |
| **현금·시재 관리** | 없음(온라인) | 시재/드로어/마감/환율 |
| 멤버십·포인트 | 유사 자산 있음 | 온·오프 통합(기존 병합) |
| 신선식품 | 콜드체인 배송 SW | 매장 유통기한/로스/저울 연동 |

**핵심 시사점**
- **POS(오프라인 계산대)는 우리가 프로덕션으로 다뤄본 적 없는 영역.** PM의 리테일·투게더 POS 경험은 요구사항 정의엔 강점이나 재사용할 개발 자산은 없음 → 신규 개발 또는 현지 POS 벤더 연동/제휴가 현실적.
- 우리 강점(온라인 주문 + B2B 정산 + 거래처 단가)을 전면에 두고, **POS·현금·매장재고는 '현지 POS 연동' 또는 Tier 3 확장**으로 리스크 관리.
- 위탁사(다자 중개) 정산 로직은 K-Market의 B2B·입점셀러·PB엔 재사용 여지가 있으나, 소매 계산대 판매와는 데이터 모델이 달라 진단에서 구분.

## G. 기술 · 인프라 · 운영

- 다국어(한/베/영), 현지 통화·주소·타임존(ICT). 지도(Goong/Vietmap)·SMS·Zalo 알림 연동.
- 현지 개발·운영·QA·CS 파트너(브릿지), 베트남 앱스토어 계정.
- 하드웨어(바코드 스캐너·저울·영수증 프린터·카드 단말) 표준·조달.

## H. 상거래 · 물류 · 회계

- 재고 회계(매장/센터), 신선 로스·폐기, 콜드체인. 환율·다통화 회계, 수입 통관.
- 반품·환불(소비자보호), 미성년 결제.
